# walking-event-app
## 0.これは何か
社内向け歩数計測アプリ

## 1. どういったものを作ろうとしているのか
システムアイの社員向けのスマホアプリ。

- このアプリをインストールしてウォーキングイベントへの参加を表明すると、
- スマホで歩数を計測してサーバに送信、
- イベント期間中の歩数を競う形としたい。

## 2. なぜ、これを作りたいのか
社員の日々の歩数及び、他の社員の歩数と比較したランキングをアプリで見える化することによって社員のウォーキングのモチベーションを上げて、社員の健康的な生活を促進したい。

## 3. どういった機能を持たせたいか

### 機能要件
1. iOS と Android の提供する機能を使用して歩数を計測して表示したい
2. 計測した歩数をサーバに送信してDBで管理して、歩数のランキングを表示したい
3. ログインの手間がかからない認証方式にしたい(ID、パスワードが基本だが、生体認証なども駆使する)

### 非機能要件
1. ウォーキングのモチベーションが上がるような UI にしたい。
2. ランニングコストをかけたくない
   - サーバ側はAWSに載せる。
   - サーバレスや DynamoDB 等が好ましい。あるいは EC2 にデータベースも含めすべて乗せる。
   - API は OpenAPI で定義する。

## 4. どういった流れでこれを作るか
ざっくり記載、詳細なのは後で作るかも

### Step1
- [health](https://pub.dev/packages/health) を使って、歩数を取得して表示する
- REST APIリクエスト(POST)を投げて、取得した歩数を送信する(サーバは構築せず、ローカルのMockサーバに投げるところまで)

### Step2
- AWSにAPIサーバを立てる
  参考: [Lambda と DynamoDB を使用して CRUD HTTP API を作成する](https://docs.aws.amazon.com/ja_jp/apigateway/latest/developerguide/http-api-dynamo-db.html)
- REST APIリクエストを投げて、取得した歩数をDynamoDBに登録する

### Step3
- 歩数のランキングを計算する方法を考える & 実装する
- ログイン画面の実装(ユーザID、パスワード方式)

### Step4
- ログイン方式の追加(OTP、生体認証など)

## 5. 実際にアプリとしてリリースするときの注意点
歩数データに対するアクセスの許可の方式がiOSとAndroidで大きく異なる。

(AndroidはGoogleアカウントとのOAuthも必要みたい)

[参考サイト](https://docs.sakai-sc.co.jp/article/programing/flutter-health.html)
